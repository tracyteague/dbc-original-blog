What's the Difference?!

Margin, Border, and Padding

January 2, 2015

Archives

Much of CSS involves positioning the "blocks", images, text, etc., of HTML in such a way that enhances the user's experience. In order to make these elements both "eye-catching" and easy to navigate, it's important to understand how to use margins, borders, and padding.

Below is a simple illustration showing the placement of and relationship between margin, border, and padding around its content:

Let's start with the border as it is the simplest concept of the three. The border is exactly what it states - a border that goes around your text, content, image, etc. A border can be manipulated in terms of style, such as solid or dotted; width, such as 5px or 10px; and/or color, such as black or blue. Moreover, the designer can choose to use the same OR different styles, widths, and colors on each side of the border by specifying which side (i.e., top, bottom, left, right).

For example, look at the image below:

The top border is pink and dotted, the bottom border is cornflowerblue and solid, and the sides do not have visible borders. Thus, the CSS code would be written as:

div {
border-top: 2px pink dotted;
border-bottom: 2px cornflowerblue
solid
}

Simple, right?

Next, we'll discuss padding. Padding, as seen in the diagram above, is the space between the actual content and the border. Thus, it is applied to the inside of the element to determine how near or far your element's content will be from the border. As with border, the specific value for padding around the content can be the same or different for each side.

Look at this image:

Here, it is visible that there is only extra space placed between the border and the content on the left side of the content (specifically, the padding is 20px for the left side of the content). Thus, the code would be written as:

div {
padding-left: 20px
}

You don't have to write "padding-right: 0px, padding-bottom: 0px, padding-top: 0px", as 0px is the default width. However, if you wish to include this for practice in the beginning, feel free to until it sinks in!

Finally, let's talk about margins. I'm sure you've heard the term "margin" when desigining in Word documents (or even when writing on looseleaf)! In CSS, the margin is the space outside of the border, and determines how far away this element is from other elements. In other words, the margin clears an area outside of the border and, unlike the border, it is completely transparent. Again, as with both border and padding, each side for the margin (i.e., top, bottom, right, left) can be the same or different.

Check out these two images:

The first image shows the outline of the margin that we implemented (highlighted in light orange), while the second shows what the image actually looks like (remember: the margin space is transparent). The code for this example would look like this:

p {
margin-left: 20px;
margin-right: 40px;
margin-top: 100px;
margin-bottom: 200px;
}

So, in sum, the order of placement starting from the content and moving outwards is padding, border, and margin. Each of these elements can be formatted or spaced differently - or the same - on each side (top, bottom, left, right).
