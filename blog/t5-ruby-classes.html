Thankfully, the creators of Ruby have provided us with several ways to organize and store our information. You can use an array to hold a simple, ordered list of items, or you can use a hash to hold a list of items with key-value pairs.

So what’s next? Classes, of course! So we can collect all of our objects and methods to group them into one simple…well…class!

WHY CLASSES?

Defining a class lets you group behaviors (methods) into convenient bundles so that you can quickly create many objects that behave essentially the same way. Classes are, at heart, a way to organize objects and methods (Thanks, David Black! You are one Well-Grounded Rubyist).

INSTANCE METHODS AND INSTANCE VARIABLES

As class methods are methods that are called on a class, instance methods are methods that are called on an instance of a class. Another way to say this is that class methods are executed in context of their arguments, while instance methods, in addition to arguments, are executed in the context of the state of the object.

Instance variables are variables that can be used by all instances of the class (rather than being specific to one method inside the single def method…end). They enable an individual object to remember its associated information or data, and can be distinguished from other variables by placing the @ sign before the variable name (e.g., @variable vs. variable).

This may all seem a bit wordy, so let’s dive into an example to further demonstrate what this means - we’ll keep it simple!

CLASSIN’ IT UP: CHECK OUT MY COOL BACKPACK!

I’m a girl, I like to shop…let’s say I bought a really cool Aztec-printed, funky-colored backpack (yes, I have a weird obsession with patterned backpacks) and I reaaally love it, so I want to leave a good review for others who are considering to buy it! (Note: there are obvious several components to setting up online reviews, but again, we’re keeping it super simple so this lesson focuses more on class structure than review details).

To make it easier to organize and store different information, let’s make a class, ProductReview:

class ProductReview
end

Inside this class, we want to create an initialize method to set up some parameters for the information that will be important. It’s often helpful to use the special method “initialize” as that method will be executed every time you create a new instance of the class.

We’ll use the name of the store, its location, the price of the backpack, and a review of the backpack:

class ProductReview
def initialize(store, location, price, review)
@store = store
@location = location
@price = price.to_f
@review = review
end
end

# See the @ signs - now those are each instance variables so that they will be accessible throughout the entire class.

Now that we have our variables, let’s add in the methods for each variable, so that the review will return the information that we want to share!

class ProductReview
def initialize(store, location, price, review)
@store = store
@location = location
@price = price.to_f
@review = review
end

def store
return "Store: " + @store
end

def location
return "Location: " + @location
end

def price
if @price < 20.00
return "$"
elsif @price > 30.00
return "$$"
elsif @price > 40.00
return "$$$"
else
return "$$$$"
end
end

def review
@review
end

end

Awesome! Let’s test it out - I bought a backpack from Urban Outfitters in New York, NY, paid $39.50 for it and loved it! We’ll create a new

tracy = ProductReview.new(“Urban Outfitters“, “New York, NY", "39.50", "I love this backpack - it's so worth the price!")

# Here, .new is a “constructor”, with the purpose of creating and returning a new instance of the class. So, we’re sending a message to our class (ProductReview) that we want to create a new review (tracy)

p tracy.store
p tracy.location
p tracy.price
p tracy.review

# Calling each method on “tracy” and printing individually to reveal my review!

Output:

Store: Urban Outfitters
Location: New York, NY
$$
I love this backpack - it's so worth the price!
