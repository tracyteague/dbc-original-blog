REVIEW OF THE BASICS

Before we get into the discussion on Ruby hashes vs. JavaScript objects, let’s do a quick review of syntax for some of the “basics “ in Ruby and JavaScript.

FUNCTIONS

Functions in JavaScript are the same as methods in Ruby, however, the syntax of declaring a function vs. declaring a method is slightly different.

# Ruby:
def hello
puts “hello”
end

// JS:
function hello() {
console.log(“hello”);
}

ARRAYS

Both Ruby and JavaScript have arrays, which are more or less logically the same; however, the Object array functions in JavaScript and the Class array methods in Ruby also have different syntax. To find the index of an element in an array:

# Ruby:
array_name.index(“element”)

// JS:
arrayName.indexOf(“element”)

RUBY HASHES VS. JAVASCRIPT OBJECTS

Now, onto our lesson! Although Ruby and JavaScript both use arrays for lists of items, sometimes, it may be easier to give each piece of data a specific name to be able to access it and retrieve it more easily. In Ruby, hashes, or key-value pairs that store information, are used for this matter, however, in JavaScript, hashes do not exist and, thus, its near equivalent – objects – would be used. Check out the example below for the general structures:

# Ruby Hash

new_hash = {
“key1” => “value1”,
“key2” => “value2”,
“key3” => “value3”,
}

new_hash[“key1”] # Accesses the current “value1” value at the “key1” key


// JavaScript Objects

var newObject = {
property1: “variable1”
property2: “variable2”
property3: “variable3”
}

newObject.property1 // Accesses the current variable “variable1” at “property1”
newObject[“property1”] // Also accesses the current variable “variable1” at “property1”

As mentioned, a Ruby hash is a collection of key-value pairs that stores information. These values can be accessed by calling on the key that corresponds to each value, and these values may be strings, integers, arrays, or other hashes. We’ll create a hash to store some information about DBC:

dbc_sea_lions = {
“Year” => 2015;
“Cohort1” => “NYC”;
“Cohort2” => “SF”;
“Cohort3” => “Chicago”;
“Cohort4” => “Columbus”;
}

p dbc_sea_lions
# use “p” to display the hash to the console
# => is a hash-rocket for the key => value pair
# used “quotation marks” for all string keys and values

JavaScript objects work similarly to Ruby hashes. The equivalent of Ruby “keys” would be “properties” in JavaScript. Each property is a variable, and each variable may also be a string, number, function, or other object. Using the above example, we’ll now store this information into a JavaScript object:

var dbcSeaLions = {
year: 2015;
cohort1: “NYC”;
cohort2: “SF”;
cohort3: “Chicago”;
cohort4: “Columbus”;
}

console.log(dbcSeaLions)
// use console.log() to print the object to the console
// colons (:) are used to separate the property and variable
// only used “quotation marks” for string variables

I hope that this brief introductory lesson helped you to notice the minor differences between Ruby hashes and JavaScript objects. Cheers, DBC Sea Lions!
