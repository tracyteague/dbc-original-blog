Array, Hash, .each

Before we get into a lesson on enumerable#map, let’s do a quick review of arrays, hashes, and the .each method as they are all related and rather “key” to enumerable.

Array: An array is an “ordered, integer-indexed collection of any object”. It looks like this:

names = [“Tracy”, “Randy”, “Denise”, “Justin”, “Nicky”]
puts names[0] #=> “Tracy”
puts names [1] #=> “Randy”
#and so on…

Hash: A hash is a “collection of key-value pairs”. So, using the above example, we could turn the array into a hash to look this:

names = {
“daughter” => “Tracy”,
“father” => “Randy”,
“mother” => “Denise”,
“brother” => “Justin”,
“pet” => “Nicky”,
}

puts names[“daughter”] # => “Tracy”
puts names[“Nicky”] # => “pet”

.each: The .each method iterates over the given enumerable, running the block for each value. A simple example looks like this:

names = [“Nick”, “James”]
names.each { | name | puts name + “ likes soccer!” }
# => “Nick likes soccer!”
# => “James like soccer!”

Enumerable

Now, onto enumerable...
A rather wordy definition: Enumerable is a “module that provides a set of methods to traverse, search, sort, and manipulate objects” and must include the method “.each” which yields successive members of this collection.

A super basic explanation: Enumerable gives you several useful ways of doing something to every element of a collection object (such as an array or hash). We use it because it's incredibly helpful and saves mucho time when building programs or websites.

Enumerable#map

Finally! Let's get to enumerable#map...
Basically, .map is a transformation method - you’re remapping your original object through some sort of transformation.

Check out this simple example:

(1..4).map { | i | i * i } #=> [1,4,9,16]

Note: “map” is non-destructive, meaning it will transform a copy of the array without altering the original. In order to make change the original array, or to be “destructive”, you would simply add an exclamation point, and would use “map!”.

Wait…so what’s difference between .map and .each?

Again, .each simply iterates over the given enumerable, running the block for each value. It discards the return value of the block and simply returns the original object it was called on:

[1,2,3].each do | i |
i + 1
end #=> [1, 2, 3]

On the other hand, .map sets the current element being iterated over to the return value of the block, and then returns a new object with those changes:

[1, 2, 3].map do | i |
i + 1
end #=> [2, 3, 4]

...so it “maps” each element to a new one using the block given.

Let’s check out one final example combining the two:

student_name = [“joe”, “devin”]

#below, we map one array to another and convert each element with the rule that will be provided

student_name.map! { |student_name| student_name.capitalize }
#now, in this example, names will be “Joe” and “Devin"

student_name.each { | name | puts name + “ is a student. “ }

#output
# Joe is a student.
# Devin is a student.

That about wraps it up!
