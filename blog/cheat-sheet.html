Enumerables

Enumerables give you many useful ways of doing something to every element of a collection object (an array or hash, for instance), which is a very common thing to do when you're building programs and websites.

An enumerable must provide a method #each in its class to yield successive members of the collection. #each calls the given block once for each element in self, passing that element as a parameter. For example: 

> [1,2,3].each { |num| print "#{num}!" }
1! 2! 3! => [1,2,3]
Two of the most commonly used enumerables are #map and #select.

Enumerables: #map and #select

Method Name Input Example Output Description
#map Takes an array
my_array = [2, 4, 6, 8, 100]
            
> my_array.map{|num| num**2 }
=> [4,16,36,64,10000]
Returns an array (Same as #collect)
#map performs a specified action on each element and returns a new array filled with whatever gets returned by the block each time it runs.
#select Takes an array or hash
my_array = [2, 4, 6, 8, 100]
            
> my_array.select{ |item| item%2==0 }
=> [2,4,6,8,100]
Returns an array #select returns a new object as an array filled with only those original items for which the block returns true.

Additional Basic Methods

Method Name Description
#any? Returns true/false and answers the question, "do any of the elements in this object pass the test in my block?". If your block returns true on any time it runs, any? will return true.
#all? Returns true/false and answers the question, "do all of the elements of this object pass the test in my block?". Every time the block runs it must return true for this method to return true.
#find Returns the first item in your object for which the block returns true.
#group_by Runs your block and returns a hash that groups all the different types of returns from that block.
#include? Returns true if any element of your block is equal to the object.
#none Returns true only if none of the elements in the object return true when the block is run.
