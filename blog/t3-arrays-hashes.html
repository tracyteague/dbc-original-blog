Arrays vs. Hashes

Both arrays and hashes are essentially collections used to store and look-up data, but what exactly is the difference between an array and a hash; what do each look like; and how do we decide which to use?

An array is an ordered collection of any object, thus, it can be viewed as more of a list. Arrays use integers as indexes to reference a specific object within this list, and begin with an index of 0 (i.e., array[0]).

Let's use an example to see what this array will look like. You’re given a list of multiple choice answers to your social studies scan-tron test, and want to locate the answer to question #9 (obviously, you made a note-to-self that you wrote “a”, but want to find out if it was actually “d”).

test_answers[a,b,c,d,a,b,c,d,a,b,c,d,a,b,c,d]

test_answer[8]

#This would print out "a"
#As a reminder, we write [8] and not [9] because the index starts at [0]!

Boom - you got it right! If only you knew the answers were in such a repetitive “abcd” order, how much time you would have saved studying…

If you’re still a bit confused, here’s another example: think of a race in the Olympics. If you’re watching track and have a list of the names of the runners in the order they placed, you could find out who came in third by calling the third-place index (array[2]) to find the associated name.

Now that we’ve covered arrays, let’s talk about hashes. Hashes are collections of pairs of information, with each pair containing a key, or name, and a value. Hashes are not ordered, and thus, one would use a key or value to retrieve the corresponding data rather than an index number. Hashes are more helpful when someone may want to look-up the name of an object or item, but the object or item wouldn’t necessarily be easy to find with a placement or order in a group.

As mentioned in Codecademy explanations, a helpful way to think of a hash is to compare it to a dictionary. A dictionary essentially has a key (the word) and a value (the definition of the word). Check out this example:

word_definition {
“happy” => “joyful”,
“sad” => “depressed”,
“content” => “satisfied”,
}

word_definition[“happy”]

#this would return “joyful”

If you’re still a bit confused, since I know you would probably never need to look up the meaning of “happy”, think about categories in Award Shows. If you wanted to look-up the winner for “Best Actor”, it would be way more to difficult to figure out how many awards were presented, the order they were presented, what number “Best Actor” was announced, etc. - you’d want to be able to find out right away by associating “Best Actor” with “Paul Rudd”! (Yes, he’s great).

I hope this explanation of arrays and hashes was clear and that you now understand the basics of what each one entails and when one might be useful over the other. Thank you for checking out my blog and feel free to reach out to me with questions or feedback!
